## 1. 프로젝트 방향성 확립

- **초기 상태**: 사용자로부터 훌륭한 프로젝트 제안서(`Proposal_20211004.pdf`)와 수업 과제 안내서(`CSE403_2025_DL_Project(1) (1).pdf`), 그리고 상세한 피드백 텍스트를 전달받음.
- **분석**:
    - 제안서는 **'AI 이미지 탐지'**라는 명확한 주제와 **CNN vs ViT 비교**라는 좋은 접근법을 가지고 있었음.
    - 과제 안내서의 핵심 평가 기준은 "단순 구현"이 아닌, **"왜 그런 결과가 나왔는지 이유를 설명하는 능력(reason about why)"**이었음.
    - 함께 제공된 피드백 텍스트는 사용자의 제안서와 과제의 핵심 평가 기준을 완벽하게 연결해주는 훌륭한 가이드라인으로 판단됨.
- **최종 결론**: 제공된 피드백 텍스트(특히 Explainability, Robustness 방향)를 따르는 것이 A+를 받기 위한 최상의 전략이라고 결론 내림.

## 2. 초기 단계 및 개발 환경 설정

- **문제**: 사용자가 구체적인 계획에도 불구하고 어디서부터 시작할지 막막함을 느낌.
- **1차 제안**: "데이터셋에서 이미지 1장 불러오기"라는 아주 작은 단계부터 시작할 것을 제안.
- **사용자 피드백**: 해당 단계가 너무 쉽다고 하여 건너뛰고, 파일 구조 구성부터 요청.
- **수행 작업**: `mkdir -p data notebooks` 명령어를 통해 프로젝트 기본 폴더 구조를 생성함.

---

## 3. 프레임워크 선정 및 데이터 파이프라인 구축

1.  **프레임워크 논의**:
    - 사용자가 "제안한 데이터셋에 어떤 프레임워크가 더 적합한지" 질문함.
    - 데이터셋 자체는 두 프레임워크 모두에 적합하지만, 프로젝트의 **분석적/연구적 성격**과 **기존 로드맵과의 호환성**을 고려할 때 **PyTorch**가 더 유리하다고 추천함.

2.  **프레임워크 결정**:
    - 사용자가 **PyTorch**로 진행하기로 결정함.

3.  **데이터 파이프라인 코드 작성**:
    - PyTorch의 표준 라이브러리인 `ImageFolder`, `DataLoader`, `transforms` 등을 사용하여 데이터 파이프라인을 구축하는 코드를 작성함.
    - 이 코드는 `notebooks/01_data_pipeline.py` 파일로 저장됨.
    - **주요 기능**:
        - 폴더 구조(`real`/`fake`)를 기반으로 자동 라벨링.
        - 모든 이미지를 모델에 맞게 224x224 크기로 변환.
        - 전체 데이터를 학습용(80%)과 검증용(20%)으로 자동 분할.
        - 배치(batch) 단위로 데이터를 효율적으로 공급할 `DataLoader` 생성.
